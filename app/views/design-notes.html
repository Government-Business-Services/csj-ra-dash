V2


Aim

V2 was an updated version of V1, based on the UR feedback received for V1. Our aim was to:

  • remove the banked score tab that was considered unnecessary
  • change some misleading content e.g. 'timeline' to 'case history'
  • change button content so users had less fear about what the button would do (case overview tab)
  • add extra contact options for the caseworker
  • add missing reference numbers
  • change the 'urgent' tag to something more meaningful
  • hide completed cases

What we did

We made the above changes to create a V2 version of the dashboard.

V1


Aim

Following on from V1, our aim was to:

  • change the look / feel from Career Site to more GDS-aligned styling
  • add a search jobs facility on the portal page itself
  • show a reuseable info section so users could see that they didn't have to repeat certain info entries every time they applied
  • let the user save a job
  • let the user set up a job alert
  • iterate the design based on V1 feedback e.g. more detailed application section, status tags, change 'Messages' to guidance panels

What we did

We redesigned the portal page with a GDS-friendly look, and then offered users two variants:

  • an iterated (from V1) portal but still based on a series of info panels
  • a more concise version, using tabs to compartmentalise the same info as on the non-tabbed version. This was for a quick user view to see if they preferred seeing less info than on the normal version.

User research

Mural board with testing notes


What we found

  • the improved applications panel was very successful, with all users understanding what each application type was
  • the status tags were also very successful, especially the red one warning of an impending deadline
  • users liked the targeted guidance panels and usually understood why they were seeing particular ones at a certain point
  • there was some debate over the placement of the details section, with some happy with it's position but others suggesting it should be in the top nav bar. Either way, the hidden details was preferred to the previous version in V1
  • users understood and liked the reuseable info section
  • all users were able to navigate the saved job and job alert journeys and had no major concerns with either
  • interestingly, a number of users said they'd prefer video content when been given guidance - in stark contrast to opinion on videos on the initial landing page prototype. This implies that while generic 'corporate' videos are not welcomed, they can be valuable when giving instructions or advice
  • a number of users felt that setting up job alerts was mentioned too many times
  • users still struggled to see how to create an account on the sign in page
  • the notification banner for a saved job might be better placed on the saved ad itself, rather than the top of the page as per standards
  • the 'new' tag was a success, although some users felt it should be more prominent e.g. in a coloured box, bolder, bigger etc
  • most users preferred the initial 'all on one page' design over the tabbed version, indicating that the amount of info on the former page is not overwhelming as was first feared
